## Tech Debt (Non-Blocking)

See YAML frontmatter `tech_debt` for the structured list. Summary:

1. **SSRF `callback_url` validation vs. internal-only deployment model** (Phase 2, accepted at T-02-01, re-flagged here) — blocks all RFC1918/loopback/link-local targets; could make WEBHOOK-01 undeliverable against a real internal network using private addressing. Worth a deliberate decision before production rollout.
2. **Webhook-loss race on Redis blip at job completion** (recommended in Phase 2's SECURITY.md, never picked up by Phase 3 or Phase 4) — a narrow, low-probability gap with zero recovery path today. Candidate for a small follow-up phase or backlog item: extend the reconciler's sweep to also catch `done`/`failed` jobs with a set `callback_url` and no `webhook_deliveries` row.
3. **Reconciler staleness soak test** (Phase 3) — real multi-minute wall-clock staleness scenarios were validated via throwaway integration tests, not a full real-time soak; consider before production rollout.
4. **Decompression-bomb / dimension-limit protection** (Phase 4, explicitly deferred as D-09) — VALID-01/02 covers format-matching only.
5. **docker-compose.yml staleness** — one real gap (missing `WEBHOOK_SIGNING_SECRET`) was found and fixed live during Phase 4's verification, caused by container images not being rebuilt for months. Recommend a one-time audit of docker-compose.yml against `.env.example`.

None of these block milestone completion — they are candidates for the next milestone's backlog or a small hardening follow-up.
