<objective>
Accept and validate a per-job `callback_url` on `POST /v1/jobs`, and surface it through the jobs domain type and repository so downstream webhook delivery (plan 02-03) can read it back from Postgres.

Purpose: This is the ingestion half of the phase's vertical slice. Without it no job carries a delivery target. It also lands the SSRF guard (D-03) at the one place a client-supplied URL enters the system.
Output: `handleCreateJob` reads/validates `callback_url`, `jobs.Job`/`CreateParams` gain `CallbackURL`, and `Repo.Create`/`Repo.Get` persist and read the existing (already-in-schema) `jobs.callback_url` column.
</objective>

## Phase Goal (MVP user story)

**As an** internal service client, **I want to** receive an HMAC-signed webhook callback when my conversion job completes, **so that** results are pushed to me without polling the job-status endpoint.

> Note: ROADMAP.md Phase 2 `**Goal:**` is stated in outcome form, not the three-slot user-story form. The mapping is unambiguous (role = internal client, capability = signed completion callback, outcome = no polling), so the story above is derived directly from it rather than invented. This plan delivers the first observable increment of that story: a client can attach and have validated a `callback_url` on job creation.

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Information Disclosure / SSRF | `validateCallbackURL` in internal/api/callbackurl.go | mitigate | Reject non-https schemes and any host resolving to loopback/RFC1918/link-local/metadata (169.254.169.254) at creation time (D-03). Blocks the service from being coerced into probing internal endpoints. |
| T-02-01b | SSRF (DNS rebinding residual) | callback_url resolution | accept | Per D-03, host is resolved+checked once at creation, NOT re-resolved per delivery attempt. Accepted residual risk: clients are internal-only services (PROJECT.md). Do not add per-delivery re-resolution. |
| T-02-04a | Tampering (error leakage) | handleCreateJob 400 response | mitigate | Rejection returns fixed string "invalid callback_url"; internal rejection reason never echoed to the client (existing no-leak convention). |
| T-02-SC | Tampering (supply chain) | new imports | accept | No package-manager installs in this plan — only stdlib `net`, `net/netip`, `net/url`. No legitimacy gate required. |
</threat_model>
