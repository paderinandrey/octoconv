## Phase Goal (MVP user story)

**As an** internal service client, **I want to** receive an HMAC-signed webhook callback when my conversion job completes, **so that** results are pushed to me without polling the job-status endpoint.

> This plan delivers the delivery machinery: signing so the receiver can trust the payload, an HTTPS deliverer proven against an httptest receiver, and the attempt/dead-letter persistence. Plan 02-03 wires these into the worker so they fire end-to-end.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| service → client callback endpoint | Outbound HTTPS POST to an external, potentially-adversarial receiver. |
| service → Postgres | Delivery-attempt audit records + dead-letter state persisted. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-02 | Spoofing / Tampering | HMAC secret usage in sign.go | mitigate | HMAC-SHA256 over timestamp+body lets receivers verify the payload originated from the service and was not altered in transit (D-01, WEBHOOK-02). secret is a param, never logged, never embedded in the payload. |
| T-02-04 | Tampering / Replay | signed payload | mitigate | Timestamp is inside the signed message and sent as X-OctoConv-Timestamp, enabling receivers to reject stale/replayed deliveries (WEBHOOK-02 replay clause). |
| T-02-06 | Repudiation | webhook_deliveries audit rows | mitigate | RecordAttempt persists status_code/attempt/delivered per attempt; MarkDeadLetter preserves exhausted deliveries for SQL investigation (WEBHOOK-04/05, D-10) — nothing silently dropped. |
| T-02-07 | Denial of Service (slow receiver) | Deliverer.hc timeout | mitigate | 10s per-attempt http.Client timeout (D-08) bounds resource hold on a hung receiver. |
| T-02-SC | Tampering (supply chain) | new imports | accept | No package-manager installs — only stdlib `crypto/hmac`, `crypto/sha256`, `net/http`, `net/http/httptest` and already-vendored `pgx`/`uuid`. No legitimacy gate required. |
</threat_model>
