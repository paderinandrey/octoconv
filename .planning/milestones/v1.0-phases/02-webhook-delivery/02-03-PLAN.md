<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| worker → client callback endpoint | Outbound signed HTTPS POST driven by asynq retries. |
| env → worker process | WEBHOOK_SIGNING_SECRET (secret material) loaded from environment. |
| worker → S3/MinIO (presign) | Fresh presigned download URL minted per attempt, embedded in payload. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-02 | Spoofing / Info Disclosure | WEBHOOK_SIGNING_SECRET loading | mitigate | Secret read once via os.Getenv with fail-fast (log.Fatalf if empty), passed as []byte into the Handler, never logged and never placed in the payload (D-01). |
| T-02-03 | Information Disclosure | presigned download_url in payload | mitigate | Regenerate a FRESH presigned URL per delivery attempt (D-09), never reuse a stale one; TTL default 6h comfortably exceeds the ~30 min retry window so a recovering/dead-lettered client can still fetch. URL only included on `done` status. |
| T-02-05 | Denial of Service (retry storm) | asynq MaxRetry + WebhookRetryDelay | mitigate | Bounded MaxRetry=6 with exponential backoff + ±25% jitter (D-05) prevents unbounded retries and thundering-herd against a shared-failing endpoint. Circuit breaker deliberately NOT added (D-06 deferred). |
| T-02-06 | Repudiation | webhook_deliveries + dead_letter | mitigate | Every attempt recorded; exhausted deliveries marked dead_letter for investigation (WEBHOOK-04/05, D-10). |
| T-02-01b | SSRF (residual) | callback_url reuse at delivery | accept | Per D-03, callback_url is trusted post-creation; delivery does NOT re-resolve/re-validate the host. Accepted residual risk for internal-only clients — do not add per-delivery re-resolution here. |
| T-02-SC | Tampering (supply chain) | new imports | accept | No package-manager installs — reuses already-vendored asynq, pgx, uuid, minio and stdlib. No legitimacy gate required. |
</threat_model>
