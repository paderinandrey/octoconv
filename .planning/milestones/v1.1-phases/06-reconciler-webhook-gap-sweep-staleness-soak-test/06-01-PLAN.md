<objective>
Add a per-job `asynq.Unique` lock to the webhook delivery queue (D-01) with a derived, worst-case TTL (D-02), mirroring the existing image-queue `ImageUniqueTTL` pattern — but correcting for the fact that `WebhookRetryDelay` carries ±25% jitter that `ImageRetryDelay` does not.

Purpose: This is the hard prerequisite for RECON-04's gap sweep. Without it, a sweep tick that re-enqueues a webhook delivery would race the asynchronous `webhook_deliveries` row write and create duplicate live tasks on every subsequent tick until the row lands. The lock makes the second (and any subsequent) enqueue within the TTL window collide safely on `asynq.ErrDuplicateTask`.

Output: `WebhookUniqueTTL`/`webhookBackoffSum`/`webhookMaxRetry`/`webhookPerAttemptTimeout` in `queue.go`, `asynq.Unique` on `NewWebhookDeliverTask`, a `webhookUniqueTTL` field on `Client`, and two new tests.
</objective>
