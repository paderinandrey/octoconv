<objective>
Create `internal/convert/dimensions.go` — a zero-dependency, hand-written binary
parser that extracts declared pixel width/height from the header of each of the
5 registered image formats (png/jpg/webp/heic/tiff), reading from a bounded 64 KiB
non-seekable stream prefix and re-stitching that prefix back onto the stream so
the full file still reaches storage. Fully test it with byte fixtures.

This is the foundational, pure, testable deliverable of Phase 7 (VALID-03). It has
no consumers yet — the handler wiring that calls `convert.Dimensions` lands in
Plan 07-02.

Purpose: Give the API a way to reject decompression-bomb uploads (small files
declaring enormous pixel dimensions) based on declared header dimensions, without
decoding pixel data, adding a dependency, or fully buffering the upload (D-01/D-03/D-07).
Output: `internal/convert/dimensions.go` + `internal/convert/dimensions_test.go`.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| uploaded file bytes -> dimension parser | Fully attacker-controlled binary; every field (sizes, offsets, box lengths, IFD offsets, marker lengths) is untrusted and may be adversarially crafted. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-01 | Denial of Service | `Dimensions()` bounded peek | mitigate | Read at most `dimPeekLen` (64 KiB) via io.ReadFull; never grow the buffer or seek. Full upload never buffered in the parser. Verified by TestDimensionsPreservesFullStream + the const value grep gate. |
| T-07-02 | Tampering / DoS | all 5 parsers + walkBoxes | mitigate | Explicit `len(b) < N` bounds check before every slice access; malformed/truncated input returns (0,0,false) / ErrDimensionsUnknown, never panics. Verified by TestDimensions*FailsClosed and ShortInputNoPanic. |
| T-07-03 | Tampering / DoS | `jpegDimensions` marker scan | mitigate | Exclude DHT(0xC4)/JPG(0xC8)/DAC(0xCC) from the SOF range so table bytes are never misread as width/height; handle standalone TEM/RSTn markers. Verified by TestDimensionsJPEG (DHT-before-SOF fixture) + grep gate. |
| T-07-04 | Tampering / DoS | `tiffDimensions` IFD offset | mitigate | Fail closed when the first-IFD offset (or any entry offset) points past the captured buffer — no seek, no unbounded read (Pitfall 2). Verified by TestDimensionsTIFF_IFDBeyondWindowFailsClosed. |
| T-07-05 | Tampering | pixel-count arithmetic | mitigate | Parsers return width/height as uint32; downstream product computed as uint64 (enforced in Plan 07-02). TestDimensionsOverflow proves no wraparound for max-uint32 dimensions (Pitfall 1). |
| T-07-06 | DoS (over-rejection) | `heicDimensions` max-ispe approach | accept | Taking the max of all ispe boxes instead of resolving the primary item via pitm/ipma can only reject more, never less — a security-conservative simplification documented in RESEARCH.md Assumptions Log A1 / Pitfall 4. |
</threat_model>
