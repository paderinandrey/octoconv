## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| accepted document content -> LibreOffice import filters | Phase 8 already gated format/macro/zip-bomb, but the document body is still parsed by soffice inside Convert |
| soffice engine output -> job-success decision | LibreOffice can exit 0 while producing empty/corrupt output; that output must not be trusted blindly |
| concurrent jobs -> shared LibreOffice profile state | Two conversions running at once must not collide on a profile lock or read each other's temp state |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-01 | Tampering | LibreOfficeConverter.Convert output handling | mitigate | validatePDF enforces non-zero size AND leading %PDF- magic bytes before Convert returns nil (D-02); exit-0-corrupt output becomes a wrapped terminal error, not a false success |
| T-09-02 | Information Disclosure / Tampering | -env:UserInstallation profile isolation | mitigate | profileDir derived from the caller's unique per-job filepath.Dir(outPath), created with os.MkdirAll(..., 0o700); no cross-job profile sharing (live-verified 4-way concurrency, RESEARCH.md Pitfall 4) |
| T-09-03 | Denial of Service | soffice process invoked on untrusted document | mitigate (partial) | ctx timeout via runCommand bounds runtime; USER nobody (plan 09-02 Dockerfile) limits blast radius; crafted-document resource-exhaustion within the timeout window is accepted residual risk (DOC-V2-05) |
</threat_model>
