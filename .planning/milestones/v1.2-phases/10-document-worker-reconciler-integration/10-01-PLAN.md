<objective>
Add the document engine-class queue plumbing that mirrors the existing image queue plumbing in `internal/queue`: a `document` asynq queue, a `document:convert` task type, a genuinely-derived (not copy-pasted) per-job unique-lock TTL and no-jitter retry schedule, and a `EnqueueDocumentConvert` producer method on `queue.Client`.

Purpose: This is the foundational layer every other Phase 10 plan builds on — the reconciler (Plan 02) routes recovery through `EnqueueDocumentConvert`, and the document worker binary (Plan 03) consumes `TypeDocumentConvert` / `QueueDocument`. The DOCUMENT_MAX_RETRY + DOCUMENT_ENGINE_TIMEOUT-derived TTL is the mechanism that bounds document retries for NON-timeout transient failures instead of letting a job retry forever; a DOCUMENT_ENGINE_TIMEOUT expiry itself is classified terminal (no retry) in Plan 10-03 per DOC-08.

Output: New exported queue constants, task constructor, retry/TTL derivation, and client producer method — all with unit tests.
</objective>
