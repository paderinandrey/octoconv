<objective>
Add the document conversion task handler (`HandleDocumentConvert`) to the worker package and a new `cmd/document-worker` binary that consumes only the `document` queue. The handler reuses the already-engine-agnostic `process()` method (which looks converters up via `registry.Lookup`, so the registered `LibreOfficeConverter` from Phase 9 handles the conversion) and mirrors `HandleImageConvert`'s completion shape, but classifies terminal-vs-transient failures through an engine-scoped `isDocumentTerminal` that treats a DOCUMENT_ENGINE_TIMEOUT expiry as TERMINAL (the image path keeps treating it as transient).

Purpose: DOC-07 (document conversions run in their own resource-isolated process/binary, independent concurrency knob) and DOC-08 (own DOCUMENT_ENGINE_TIMEOUT; a timeout classified as a TERMINAL failure — MarkFailed + SkipRetry, no retry — a deliberate divergence from the image engine that is the only reliable backstop against LibreOffice's documented hang-on-bad-input behavior; NON-timeout transient failures stay bounded by DOCUMENT_MAX_RETRY so nothing is retried forever). Per D-05 the reconciler sweep stays solely in cmd/worker; per D-06 this binary produces but never consumes webhook tasks.

Output: A working document-worker binary and handler, unit-tested for terminal (including timeout=terminal) vs transient classification, with the image-path classifier left unchanged.
</objective>
