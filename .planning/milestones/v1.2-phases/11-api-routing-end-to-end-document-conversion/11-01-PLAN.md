<objective>
Make the engine-routing decision in `handleCreateJob` a real, content-derived, testable piece of code instead of the hardcoded `Engine: engineImage` / `EnqueueImageConvert` call. Each converter self-describes its engine class via a new `Engine()` interface method (D-01); the registry exposes `EngineFor` (D-02); `handleCreateJob` uses it to set the job's engine and pick the correct queue. Also lock in (via test) that documents already skip the image-only pixel-dimension check (D-06).

Purpose: This is the last hardcoded image-only assumption in the request path. Once routed correctly, an accepted office document reaches the document queue/worker/engine that Phases 8-10 already built, with zero further plumbing.
Output: Engine-aware routing code + unit tests proving image uploads still hit the image queue and document uploads hit the document queue.
</objective>
