<objective>
Write the project's first true end-to-end test: a committed, env-gated Go test in a new `internal/e2e` package that drives the full document pipeline over real HTTP against a running docker-compose stack (D-03/D-04/D-05). It covers all 6 document format pairs (docx/xlsx/pptx/odt/ods/odp → pdf) via `POST /v1/jobs` → poll `GET /v1/jobs/{id}` → verify the presigned download URL returns valid PDF bytes, and folds webhook delivery (SC#3) into one of those pairs. It also adds an E2E-only compose override so the actual run (Plan 11-03) is a one-command invocation.

Purpose: SC#4 demands a LIVE proof that documents flow through the same upload→convert→download→webhook infrastructure as images, matching the project's "live e2e verified" bar. This plan writes and commits the test; Plan 11-03 runs it against live infra.
Output: `internal/e2e/e2e_test.go` (self-skipping without env), 6 real fixtures in `internal/e2e/testdata/`, and `docker-compose.e2e.yml`.
</objective>

The salt MUST match the running API's API_KEY_SALT or auth will fail — document that in a comment.

    (3) `postJob(t, baseURL, apiKey, filename string, data []byte, target, callbackURL string) string` — build a multipart body (fields file/target and, when non-empty, callback_url), POST to `baseURL+"/v1/jobs"` with `Authorization: ApiKey <apiKey>`, assert 202, and return the `job_id`.

    (4) `pollUntilDone(t, baseURL, apiKey, jobID string, timeout time.Duration) map[string]any` — loop GET `/v1/jobs/{id}` on a ~2s interval until `status` is `done` or `failed` or the deadline passes; `t.Fatalf` on `failed` (surfacing error_code/error_message) or timeout; return the terminal body. Use a generous default timeout (LibreOffice cold start is slow — at least 120s).

    (5) `startWebhookReceiver(t) (callbackURL string, received <-chan webhookHit)` — create an httptest server bound to ALL interfaces (not loopback) so the container can reach it: build a `net.Listener` via `net.Listen("tcp", "0.0.0.0:0")`, wrap with `httptest.NewUnstartedServer(handler)`, replace `.Listener`, then `.Start()`. The handler captures the X-OctoConv-Signature/X-OctoConv-Timestamp headers and body and sends them on a buffered channel, then returns 200. Build the returned callbackURL from `E2E_WEBHOOK_HOST` + the listener's port (e.g. `http://host.docker.internal:<port>/webhook`), NOT the httptest default 127.0.0.1 URL. Register `t.Cleanup` to close the server.

    Follow the repo test style: plain `if`/`t.Fatalf`, no third-party assertion libs. Since this is the first HTTP-polling E2E suite, `t.Run` MAY be used for the 6-pair table (Task 3) as a deliberate, commented deviation from the codebase's no-subtests convention — note the deviation in a file-level comment.
