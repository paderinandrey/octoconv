<objective>
Generalize LibreOfficeConverter from "always ->pdf" to arbitrary intra-family document pairs, and validate every non-PDF output structurally before the job can be marked done. This delivers CONV-01 (docx<->odt, xlsx<->ods, pptx<->odp via the existing engine) and CONV-02 (structural output validation, terminal on mismatch).

Purpose: The v1.2 converter baked in three hidden "output is always .pdf" assumptions (produced-file extension, `--convert-to pdf:`, `validatePDF`). Cross-format conversion invalidates all three. The fix reuses the already-tested `convert.SniffContainer` primitive as the output validator, mirroring the input-side guarantee ("output validated by the same sniff that validates input", D-03) with zero new validation code.
Output: A target-aware `LibreOfficeConverter` + a `validateDocumentOutput` dispatcher, coupled in one commit with the worker's terminal-error classification so mismatched output fails terminally.
</objective>
