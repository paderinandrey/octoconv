<objective>
Reject OLE-CFB uploads (legacy binary .doc/.xls/.ppt and password-protected OOXML — both share the identical 8-byte header) with a clear, diagnostic 422 before any S3 write, so these files stop reaching the worker and dying as an opaque soffice timeout. Delivers SAFE-01.

Purpose: Both file classes begin with `D0 CF 11 E0 A1 B1 1A E1` and are unconvertible; a single fail-closed magic check (per-milestone scope: NOT distinguishing legacy vs encrypted — that is v2 DOCV3-02) gives internal clients an actionable rejection instead of a mystery hang.
Output: A new single-purpose `convert.IsOLECFB` detector and a third fail-closed branch in `handleCreateJob`, both mirroring the existing macro/zip-bomb inline-reject pattern (not the registry sniff path).
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client -> API (multipart upload) | Untrusted file bytes cross into the service; content is validated fail-closed before any S3/Postgres write. |
| unconvertible input -> worker/engine | Without pre-flight rejection, a CFB file reaches soffice and dies as an opaque timeout — the defect SAFE-01 closes. |

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-13-04 | Denial of Service | soffice engine fed an unconvertible CFB file | mitigate | IsOLECFB rejects CFB uploads 422 at the API boundary before S3/queue, so no CFB file consumes a worker slot / DOCUMENT_ENGINE_TIMEOUT (SAFE-01). |
| T-13-05 | Information Disclosure | 422 error text | mitigate | The message names format classes only (legacy/binary, password-protected) and a generic remedy; it echoes no file bytes, no internal paths, and no decryption attempt is made (distinction deferred to v2). |
