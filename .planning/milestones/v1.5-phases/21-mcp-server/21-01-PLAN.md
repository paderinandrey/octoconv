<wire_contract>
<!-- Extracted from the reference above and internal/e2e/e2e_test.go. The executor implements against THIS, no codebase exploration needed. -->

Auth header (internal/auth/middleware.go): `Authorization: ApiKey <key>` (scheme token is exactly "ApiKey").

POST /v1/jobs — multipart/form-data fields:
  - `file`   (form file; filename matters — its extension is the declared source format)
  - `target` (string; e.g. "jpg") — mutually exclusive with `preset`
  - `preset` (string) — mutually exclusive with `target`/`opts`
  - `opts`   (JSON string; optional)
  - `callback_url` (optional; unused by MCP)
  Response 202: `{"job_id": "...", "status": "queued", "opts"?: {...}}`
  Both target+preset supplied → 422 `{"error":"specify either preset or target/opts, not both"}`
  Neither supplied → 400 `{"error":"missing target format"}`
  Unsupported pair → 422 `{"error":"unsupported conversion: <from> -> <to>"}`

GET /v1/jobs/{id} — Response 200: `{"job_id","status","opts"?,"download_url"? (when status==done),"error_code"?,"error_message"? (when status==failed)}`
  Terminal statuses: "done", "failed". Non-terminal: "queued", "active".
  `download_url` is a presigned S3 URL — self-authorizing, fetch with NO Authorization header.

GET /v1/formats — Response 200: `{"engines": {"<class>": {"pairs": [["png","jpg"], ...]}}}`

GET /v1/presets[?all=true] — Response 200: JSON array of
  `{"name","version","scope","operation","target_format","options","description","is_active","created_at","updated_at"}`
</wire_contract>

<dial_redirect_contract>
<!-- From internal/e2e/e2e_test.go downloadClient() (~lines 703-724). Mirror this EXACTLY. -->
<!-- The presigned V4 signature covers the Host header, so rewriting the URL's host would break
     the signature — but dialing a DIFFERENT address under the ORIGINAL Host does not. -->

When OCTOCONV_S3_DIAL_ADDR is empty (production default): use a plain *http.Client with an
explicit Timeout — no custom Transport, direct dial to the URL host.

When OCTOCONV_S3_DIAL_ADDR is set (e.g. "127.0.0.1:9100"): use an *http.Client whose
Transport is an *http.Transport with a DialContext that IGNORES the addr argument and dials
the configured OCTOCONV_S3_DIAL_ADDR instead, via a net.Dialer.DialContext. The request URL and
its Host header are left UNTOUCHED. Both branches carry an explicit Timeout; only Transport/DialContext differ.
</dial_redirect_contract>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| MCP client (LLM agent) → tool input | Agent-supplied `path`, `target`, `preset`, `opts` are untrusted (agent reasoning can be prompt-injected) |
| internal/mcpserver → OctoConv API | Carries a real client API key in the `Authorization` header — must never surface downstream |
| API presigned URL → local filesystem | Server-supplied filename drives a local write — path containment required |
| go.mod → module supply chain | First new direct dependency since v1.0 |
