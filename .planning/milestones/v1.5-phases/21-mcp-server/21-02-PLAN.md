<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Agent tool call → convert_file/download_result | `path`/`filename` are agent-controlled; validated + sanitized before FS/HTTP use |
| Upstream API error → tool result | API 4xx/5xx text is forwarded to the agent; must stay no-leak and never become a protocol error |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-21-02 | Tampering | download_result / convert_file path | mitigate | filepath.Abs+Clean + regular-file check on input; sanitizeBasename + OUTPUT_DIR-only writes on output (D-09) |
| T-21-04 | Denial of Service | convert_file blocking past idle window | mitigate | NotifyProgress on every poll tick + CONVERT_TIMEOUT bound → graceful get_job_status handoff (D-04, Pitfall 1) |
| T-21-06 | Information Disclosure | tool result payload | mitigate | Results carry presigned_url + local_path only; no-inline-bytes unit test (D-07) |
| T-21-07 | Repudiation/Error-handling | isError vs protocol error | mitigate | API failures → CallToolResult{IsError:true} with nil Go error, verbatim no-leak API text (D-11) |
| T-21-05 | Elevation of Privilege | conversion authorization | accept | Zero-privilege: API is the single enforcement authority; tools only forward (D-02) |
</threat_model>
