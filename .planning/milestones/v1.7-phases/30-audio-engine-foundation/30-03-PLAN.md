<objective>
Implement `AudioConverter` — the fourth engine class — as a standalone `Converter` whose `Convert()` runs the two-stage `ffmpeg` normalize → `whisper-cli` transcribe pipeline inside one caller-supplied timeout context, mapping the four output targets (txt/srt/vtt/json) onto whisper-cli's own output flags via the `Pair` mechanism, with `target=json` emitting segment- and word/token-level timestamps (AUD-02). Verify the JSON schema live against the pinned whisper-cli v1.9.1 binary.

Purpose: This is the transcription core of the audio engine. It must be fully built and testable standalone against the local binary (phase goal) — proving the novel two-stage pipeline and the SEED-001-forward JSON timestamp contract — WITHOUT wiring it into the live API/queue/worker path.

Output: `whisper.go` (AudioConverter) + `whisper_test.go` (live-gated); MIMEType cases for the four audio output formats. AudioConverter is deliberately NOT registered into `convert.Default` — see the registration note in Task 1.
</objective>
