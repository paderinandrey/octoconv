<objective>
Wire the audio class into the two request/recovery routing paths: the API's content-detection chain (SniffAudio, chained correctly off `rest` to avoid the silent 12-byte truncation trap), the opts-parsing switch (a dedicated `EngineAudio` case so `{"language":...}` is validated by `ParseAudioOpts` instead of being rejected as invalid `DocOpts`), the enqueue switch, and the reconciler's stranded-job routing switch — plus the SC4 zero-spurious-recovery proof. Both the API `Enqueuer` interface and the reconciler `enqueuer` interface (two independent declarations) gain `EnqueueAudioConvert`, and both test doubles gain the method.

Purpose: This plan makes the audio surface LIVE and client-facing. It closes two confirmed integration bugs (opts mis-routing, upload truncation) that a naive "copy the pattern" pass would silently introduce.
Output: SniffAudio splice, opts+enqueue switch cases, both Enqueuer interfaces, reconciler routing, SC4 test.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client upload bytes → SniffAudio magic-byte peek | untrusted container bytes classified; re-stitched reader must preserve full file |
| client opts JSON → ParseAudioOpts → engine argv | audio opts must go through the closed-allowlist AudioOpts path, not DocOpts |
| stranded jobs.engine value → reconciler routing switch | engine string drives which queue a recovered job re-enters |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-31-02 | Tampering (data integrity) | SniffAudio peek-chaining silently truncating uploads | mitigate | Chain SniffAudio off `rest` (Sniff's full-file re-stitch), not `file` (cursor already advanced 12 bytes); byte-integrity test asserts stored object == uploaded bytes (RESEARCH.md Pattern 2). |
| T-31-04 | Tampering (opts injection surface) | audio opts silently validated as DocOpts | mitigate | Dedicated EngineAudio opts case routes through ParseAudioOpts + ValidateAudioApplicability (closed language allowlist, T-30-05), instead of the strict DocOpts reject that would 422 every audio opts request (RESEARCH.md Pattern 3). |
| T-31-09 | Denial of Service | unroutable stranded audio job / double-processing | mitigate | sweep() EngineAudio case (no longer hits fail-closed default); AudioUniqueTTL + asynq.ErrDuplicateTask prevent double-processing — SC4 test proves zero spurious recoveries under repeated ticks. |
| T-31-10 | Spoofing / mis-routing | non-audio ISOBMFF (MP4 video) sniffed as audio | accept | SniffAudio's m4a matcher was hardened in Phase 30 (T-30-04: m4aBrands reduced to M4A /M4B only, MP4-video ftyp rejected); this phase only invokes the already-hardened detector. No new detection logic added. |
| T-31-SC | Tampering (supply chain) | go module installs | accept | No new external packages this phase (RESEARCH.md Package Legitimacy Audit N/A). |
