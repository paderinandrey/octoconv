<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| container → shared CPU quota | Over-threading whisper-cli beyond the cgroup quota causes CFS throttling / starves co-scheduled work |
| env/config → engine argv | AUDIO_THREADS is operator-set process env, never client-derived |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-32-04 | Denial of Service | whisper-cli defaulting to host core count under a cgroup CPU quota (throttling, unpredictable wall time) | mitigate | Explicit `-t <n>` sized to `CgroupCPULimit()` (floor of quota/period), env override, NumCPU fallback |
| T-32-05 | Denial of Service | `AUDIO_THREADS` set larger than the quota (oversubscription) | accept | Operator escape hatch; documented in .env.example that it should match the RTF-measured thread count; floor-not-ceil default is the safe path |
| T-32-06 | Tampering | Client bytes reaching the `-t` argv element | mitigate | `-t` value is an int resolved from env/cgroup only (`strconv.Itoa`), never from request input (env-only-in-main convention) |
</threat_model>
