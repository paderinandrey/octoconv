| operator env → retry/timeout budget | `AV_MAX_RETRY` / `AV_ENGINE_TIMEOUT` are operator-controlled and size the resource ceiling for the whole engine class |

| T-35-06 | Denial of Service | unbounded/mis-scheduled retry via `RetryDelayFunc` default fallthrough | mitigate | `case TypeAVConvert` added in the same edit as `TypeAVConvert`; `TestRetryDelayFuncRoutesAVConvert` fails if the arm is removed |
