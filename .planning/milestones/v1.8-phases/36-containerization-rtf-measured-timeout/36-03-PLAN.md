<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| operator env → every queue.NewClient() process | AV_ENGINE_TIMEOUT/AV_MAX_RETRY size AVUniqueTTL in every enqueuing process; divergence duplicates processing |
| derived timeout → 900s reconciler cap | AV_ENGINE_TIMEOUT must stay under RECONCILER_ACTIVE_STALE_AFTER or the reconciler re-enqueues mid-transcode |
| CI cache scope → build reproducibility | av-worker bake scope must be isolated so a poisoned cache can't cross-contaminate other images |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-36-08 | Denial of Service | AVUniqueTTL divergence from env mismatch | mitigate | IN-02: AV_ENGINE_TIMEOUT + AV_MAX_RETRY identical on all 8 queue-client services; grep-count == 8 + single-value-across-all assertion pins parity |
| T-36-09 | Denial of Service | AV_ENGINE_TIMEOUT breaching the 900s reconciler cap | mitigate | Provisional 600s stays under 900s; RECONCILER_ACTIVE_STALE_AFTER left unchanged; NO-GO lever (lower AV_MAX_DURATION_SECONDS) is finalized at the Plan 04 checkpoint, never by raising the cap |
| T-36-07b | Elevation of Privilege | av-worker service privilege | mitigate | Service inherits USER nobody from Dockerfile.av-worker (Plan 02, T-36-07); compose adds no privileged/cap-add directive |
| T-36-SC | Tampering | CI bake cache scope | mitigate | av-worker uses an isolated `scope=av-worker` gha cache, mirroring every other image; no shared scope |
</threat_model>
