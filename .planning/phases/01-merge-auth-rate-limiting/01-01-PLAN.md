<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| operator shell → cmd/manage-clients → Postgres | Raw high-entropy key generated in process memory, printed once to stdout, only its salted digest persisted |
| Postgres at rest | Stored key material must be non-reversible even if the DB is leaked |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Information disclosure | cmd/manage-clients key output | mitigate | Raw key printed exactly once via `fmt.Println`; never passed to `log.*`; only salted SHA-256 digest reaches Postgres (AUTH-04) |
| T-01-02 | Spoofing | auth.GenerateKey | mitigate | 32 bytes from `crypto/rand`, base64url — high entropy, not guessable/enumerable |
| T-01-03 | Information disclosure | clients table at rest | mitigate | Keys stored as `sha256(API_KEY_SALT ‖ key)` hex digest; server-side pepper lives outside the DB, so a DB-only leak yields non-reversible, non-precomputable digests (PITFALLS Pitfall 6) |
| T-01-SC | Tampering | go.mod dependencies | accept | Plan 01 adds NO new module dependency (stdlib crypto/sha256, crypto/rand, encoding only); nothing to vet |
</threat_model>
