## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client→DB (engine column) | The `engine` value written to `jobs` is server-derived (EngineFor), never client-supplied; the CHECK constraint is a defense-in-depth backstop against a code bug inserting an invalid engine string. |
| reconciler→queue | The reconciler re-enqueues stranded rows; an engine value it cannot route must fail closed, not silently drop. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-15-01 | Tampering | jobs.engine column | mitigate | CHECK constraint restricts engine to a fixed allow-list incl. 'html'; a corrupt/hand-inserted row with an unknown engine is rejected at the DB boundary. |
| T-15-02 | Denial of Service | reconciler sweep | mitigate | Explicit `case convert.EngineHTML` prevents an html job from being silently skipped by the fail-closed default and never recovered (would otherwise strand jobs forever). |
| T-15-03 | Repudiation | engine-class literal drift | accept | EngineHTML single-source-of-truth const eliminates duplicated "html" literals; DEBT-02 pattern enforced, low residual risk. |
</threat_model>
