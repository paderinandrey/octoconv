<objective>
Build the internal/convert half of the html engine: the fail-closed HTML content sniff, the closed validated print-opts struct that emits a server-constant CSS `@page` block (NOT chromium CLI flags — RESEARCH's critical Pattern 1 correction of D-06), and the ChromiumConverter that injects that CSS into a worker-built copy of the input and shells out to `chromium-headless-shell` with the layered network-block flags.

Purpose: This is the security-critical core of the phase. It carries Phase 14's "client bytes never reach the engine" invariant into a new attack surface (untrusted HTML rendered by a removed-sandbox browser). Every print option is a compile-time CSS string selected only by a validated enum; the network-block flags are the primary SSRF containment.

Output: A registered `{html→pdf}` converter, fully unit-tested (strictness, injection resistance, CSS-from-constants, sniff), whose exact chromium flags and CSS honoring are LIVE-CONFIRMED in Plan 04 before acceptance.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client HTML → CSS builder | Untrusted HTML bytes and untrusted opts JSON cross into the worker; buildPrintCSS and ParseHTMLOpts are the choke point that must never let a client byte become an executable/output-affecting string outside the validated enum. |
| client HTML → chromium renderer | The rendered.html (client content + server CSS) is handed to a removed-sandbox browser; network-block flags are the primary containment (proven live in Plan 04/05). |
| worker fs → chromium file:// | chromium is launched with file:// access; untrusted HTML could reference other file:// paths (residual risk verified live in Plan 04). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-15-04 | Tampering (injection) | buildPrintCSS / ParseHTMLOpts | mitigate | CSS built ONLY from validated enum via server-constant map; unknown fields + out-of-range rejected (checkStrictObject + DisallowUnknownFields); `!important` prevents client CSS overriding print geometry. Injection unit test asserts no client byte reaches CSS. |
| T-15-05 | Information Disclosure | SSRF via rendered resource refs | mitigate | argv carries `--proxy-server=127.0.0.1:9` (IP-literal + external), `--host-resolver-rules=MAP * ~NOTFOUND` (DNS), `--proxy-bypass-list=<-loopback>` (loopback literals). Live canary proof in Plan 04/05. |
| T-15-06 | Elevation of Privilege | removed-sandbox renderer | mitigate | `--blink-settings=scriptEnabled=false` disables the JS engine (removes a whole exploit class under forced `--no-sandbox`, Pitfall 10); container isolation is the secondary control (Plan 04). |
| T-15-07 | Information Disclosure | file:// residual read of sibling files | accept (pending Plan 04 live test) | Same-UID nobody temp dirs do not isolate concurrent jobs; internal-only trust model. Live-verified and logged as explicit residual risk in Plan 04 (matches DOC-V2-05 precedent). |
| T-15-08 | Spoofing | binary masquerading as .html | mitigate | LooksLikeHTML fail-closed content check (used by the API branch in Plan 03) rejects non-UTF-8/NUL/marker-less content. |
</threat_model>
