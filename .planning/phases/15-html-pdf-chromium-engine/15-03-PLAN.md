<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client→API (multipart) | Declared source/extension and opts JSON are attacker-controlled; the sniff branch + engine-keyed opts validation must fail closed before any storage/DB write. |
| API→queue | Only a validated, format-pair-checked html job is enqueued to the html queue; the fail-closed default catches routing bugs. |
| queue→worker | The task payload carries only job_id; all detail (incl. opts) is re-read from Postgres and strictly re-parsed (D-10). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-15-09 | Spoofing | .html extension over binary content | mitigate | LooksLikeHTML gate → detected stays "" → 422 unrecognized-content before storage; handler test asserts Upload not called. |
| T-15-10 | Tampering | opts smuggling across engines | mitigate | Engine-keyed dispatch: html jobs go through ParseHTMLOpts+ValidateHTMLApplicability only; a document-only opt on html → 422. Persisted value is the normalized struct, never raw bytes (D-08). |
| T-15-11 | Denial of Service | stuck chromium render | mitigate | isHTMLTerminal classifies HTML_ENGINE_TIMEOUT expiry terminal → no retry storm; worker concurrency cap bounds concurrent renders. |
| T-15-12 | Tampering | corrupt jobs.options column | mitigate | HandleHTMLConvert strict re-parse (HTMLOptsFromMap) → MarkFailed terminal on garbage, no silent bad render. |
</threat_model>
