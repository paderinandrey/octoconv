<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| container→network egress | The chromium-worker container renders attacker-influenced HTML; compose network config is the secondary containment layer behind the CLI network-block flags. |
| container fs → chromium (file://) | file:// access inside the container is an attacker-directed read primitive; same-UID nobody temp dirs do not isolate concurrent jobs. |
| build → apt package | chromium-headless-shell is pulled from the Debian archive at image build time. |
| chromium fork tree → PID 1 | chromium forks zygote/GPU/renderer subprocesses; without a reaper they can zombie on timeout-kill. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-15-05 | Information Disclosure | SSRF via rendered refs (external IP + loopback literal + file://) | mitigate | Layered CLI flags (dead proxy + `--proxy-bypass-list=<-loopback>` + NXDOMAIN resolver) confirmed live (items 4/5); file:// residual read tested (item 6) and logged as explicit residual risk. Full canary proof in Plan 05. |
| T-15-06 | Elevation of Privilege | removed-sandbox renderer (`--no-sandbox` forced under USER nobody, Pitfall 10) | mitigate | Compensating controls: JS disabled (scriptEnabled=false, confirmed item 2) + container network egress restriction + no interactive mode; USER nobody + resource limits bound blast radius. |
| T-15-13 | Denial of Service | zombie subprocesses / /dev/shm exhaustion (Pitfall 12/C) | mitigate | tini PID-1 reaper (necessity confirmed item 7) + `--disable-dev-shm-usage` flag AND compose `shm_size` (belt-and-suspenders per Pitfall C); runCommand process-group SIGKILL on timeout. |
| T-15-SC | Tampering | chromium-headless-shell apt install | accept | Official Debian bookworm archive package (Package Legitimacy Audit: Approved; slopcheck N/A for OS packages, not a language-ecosystem registry). No npm/pip/cargo installs in this phase. Version pinned/verified via `apt-cache policy` at build. |
| T-15-14 | Denial of Service | render-bomb HTML (huge DOM) | accept | Bounded by HTML_ENGINE_TIMEOUT (terminal) + HTML_WORKER_CONCURRENCY cap + MAX_UPLOAD_BYTES; matches accepted residual DOC-V2-05 (no active complexity anti-DoS this milestone). |
</threat_model>
