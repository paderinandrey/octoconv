    4. Exported struct `PGAdvisoryLock` holding an unexported `pool *pgxpool.Pool` and an unexported `conn *pgxpool.Conn` (the dedicated long-lived connection). Constructor `func NewPGAdvisoryLock(ctx context.Context, pool *pgxpool.Pool) (*PGAdvisoryLock, error)` acquires one connection via `pool.Acquire(ctx)` and keeps it — never Release() during process life (D-02: lock release must be tied to process/session lifetime, not returned to the pool where it would be recycled). Return the wrapped error with the fmt.Errorf("...: %w", err) convention on acquire failure.

    5. Method `func (l *PGAdvisoryLock) TryAcquire(ctx context.Context) (bool, error)`: if `l.conn` is nil (lost on a prior tick), lazily re-acquire it via `l.pool.Acquire(ctx)`; on failure return (false, wrapped error). Then run `l.conn.QueryRow(ctx, "SELECT pg_try_advisory_lock($1)", advisoryLockKey).Scan(&acquired)`. On a query/scan error, Release the (now-suspect) conn back and set `l.conn = nil` so the next tick re-acquires, and return (false, wrapped error) — fail-safe. On success return (acquired, nil). Note: once pg_try_advisory_lock returns true on this session it stays held for the session's life; subsequent calls on the SAME session return true again (Postgres allows re-acquire by the owning session), which is the intended steady state for the leader.

    6. Method `func (s *Sweeper) RunWithLock(ctx context.Context, lock AdvisoryLock)`: identical ticker structure to Run (NewTicker(s.cfg.SweepInterval), defer Stop, select on ctx.Done()/ticker.C), but on each tick call `ok, err := lock.TryAcquire(ctx)`; if `err != nil || !ok` continue to the next tick WITHOUT sweeping (fail-safe / not-leader skip); only when `ok && err == nil` call `s.sweep(ctx)`. Follow the existing best-effort discipline: do not crash, do not log inside the loop (visibility is job_events per existing convention).

    Keep the existing Run method as-is (existing soak tests call it). Do NOT change NewSweeper's signature.
