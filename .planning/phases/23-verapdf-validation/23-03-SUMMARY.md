## Decisions Made

See `key-decisions` in the frontmatter for full rationale; condensed here:

1. **XMP corruption method** — `pdfaid` -> `pdfXid` (length-preserving, all 5 occurrences) on a genuine soffice-produced PDF/A-2b export, empirically confirmed non-compliant inside the pinned image (not assumed) — breaks ISO 19005-2:2011 clause 6.6.4's required namespace prefix.
2. **Shim argv parsing** — last argv element = input path, element after `--outdir` = workDir, matching `internal/convert/libreoffice.go`'s exact invocation shape; dry-run tested locally before wiring into compose.
3. **Compose override shape** — a bare `volumes:` key under the `document-worker` service in `docker-compose.e2e.yml`, verified to merge cleanly onto the base compose file via `docker compose config`.
4. **D-10 build-time A/B methodology** — two controlled `--no-cache` local builds against the pre-23-01 and current Dockerfiles from the identical source tree, rather than an uncontrolled CI-history citation (CI itself uses GHA caching, not directly comparable to a from-scratch build).
5. **Shared-stack reuse for the live gate** — per this execution's `parallel_execution` meta-instructions (`-p octoconv`, `compose stop` not `down -v`), the already-running shared compose project was rebuilt and reused rather than spinning up an isolated stack; this explicitly and correctly overrides the plan's own `<done>` text mentioning `down -v`.
