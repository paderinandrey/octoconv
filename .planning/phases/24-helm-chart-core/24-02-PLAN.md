## DECISION REFINEMENT (surfaced, not silent) — migrate hook phase
D-05 specifies migrate as a `pre-install,pre-upgrade` hook. On a FRESH install this deadlocks: the `postgres` Service is a main-release resource that does not exist during the pre-install phase, so a pre-install migrate Job (or its wait-for-postgres initContainer) can never reach a database. Honoring D-05's INTENT (a single idempotent migration via a hook Job, run once before app pods consume the DB) requires the hook to run AFTER the main resources are created. Resolution: run BOTH hook Jobs as `post-install,post-upgrade` (migrate hook-weight "0", createbucket hook-weight "1"), each with an initContainer that waits for its dependency to be Ready. This matches the phase D-05 already assigns createbucket (`post-install/post-upgrade`) and keeps upgrades data-safe (stateful deps stay normal main resources, never hook-deleted). Record this refinement in the SUMMARY for operator awareness.
