<objective>
Add the `EngineAudio` engine-class constant and the validated-opts layer for audio transcription: `AudioOpts{Language, Translate}` parsed through the same closed-struct, strict-JSON, allowlist discipline the document and HTML engines already use (AUD-03/OPTS-01 precedent), with an injection test proving client bytes never reach the whisper-cli argv.

Purpose: Client-supplied transcription options are untrusted. The language code must be validated against a closed allowlist and only ever passed as an argv slice element — establishing the injection-safe pattern from day one so a future phase extending AudioOpts (e.g. a model selector) follows it rather than reinventing it. The `EngineAudio` constant is the single source of truth `AudioConverter.Engine()` (Plan 03) and the future routing/queue layer (Phase 31) will reference.

Output: `EngineAudio` const in convert.go; `audioopts.go` (+test). No API/queue/worker/registry wiring (scope fence) — AudioConverter is NOT registered this phase.
</objective>
