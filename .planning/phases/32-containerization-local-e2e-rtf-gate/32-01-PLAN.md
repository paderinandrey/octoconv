<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| build host → upstream (HuggingFace, GitHub) | Model + source pulled over network at build time; untrusted until pinned |
| runtime container → host | Worker shells out to whisper-cli/ffmpeg on untrusted audio input; runs as nobody |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-32-SC | Tampering | HuggingFace `ggml-base.bin` fetch (model could be swapped between research-time capture and build-time fetch) | mitigate | `sha256sum -c` fail-closed check in the whisper-build stage (build aborts loud on mismatch); re-verify hash live at execution (Assumption A1) |
| T-32-01 | Tampering | `git clone --branch v1.9.1` (a tag can theoretically be force-moved upstream) | accept | Matches the project's existing tag-based pinning convention (verapdf/cli:v1.30.2, asynqmon:0.7.2 — none commit-SHA pinned); low likelihood for a released widely-used repo; not a new gap this phase introduces |
| T-32-02 | Elevation of Privilege | Runtime container executing whisper-cli/ffmpeg on adversarial audio | mitigate | `USER nobody` unprivileged runtime (unchanged pattern across all 5 existing worker Dockerfiles) |
| T-32-03 | Denial of Service | Compiled binary using `-march=native` and SIGILLing / mis-threading on runtime host | mitigate | `-DGGML_NATIVE=OFF` (portable build); explicit `-t` thread ceiling wired in Plan 02 |
</threat_model>
