<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client JSON opts → AVOpts parse | Untrusted client bytes are strict-parsed and allowlist-validated before they can influence any ffmpeg argv |
| uploaded file → ffprobe resolution probe | Untrusted container metadata is read by a hardened, protocol-whitelisted ffprobe before the expensive decode |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-34-05 | Tampering | ParseAVOpts (client bytes reaching argv) | mitigate | checkStrictObject + DisallowUnknownFields + closed enum/codec/timecode validation; client values map server-side to fixed constants (resolution enum, codec allowlist), never concatenated into argv strings |
| T-34-06 | Elevation of Privilege | Filtergraph/source-filter injection via a free-form opt | mitigate | AVOpts is a closed typed struct — no field carries a raw client string past the parse boundary; no `-vf`/`-filter_complex` argument is derived from client bytes |
| T-34-07 | Denial of Service | Multi-axis decode bomb (huge resolution) | mitigate | probeVideoStream + EnforceMaxResolution reject an over-ceiling declared resolution before the expensive stage; complements the reused EnforceMaxDuration axis |
| T-34-08 | Information Disclosure | SSRF/LFI via container-embedded remote reference during the resolution probe | mitigate | ffprobeStreamArgs carries `-protocol_whitelist file,crypto` and a `file:`-prefixed path (AVE-02) on the new probe |
| T-34-09 | Tampering | x265 path silently inheriting x264's CRF constant (AVO-03 footgun) | mitigate | Two distinct compile-time constants (x264DefaultCRF/x265DefaultCRF) with a test asserting they differ |
| T-34-SC | Tampering | npm/pip/cargo installs | accept | No package-manager dependencies added this phase; ffprobe is an OS binary invoked via runCommand |
</threat_model>
