<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Redis task payload → worker | Task payloads are trusted only as a job ID; all job detail is re-read from Postgres (system of record) |
| stored job.Opts → converter | Opts persisted at create time are strictly re-parsed before use, so a tampered row cannot reach ffmpeg argv |
| ffmpeg stderr → job failure detail | Subprocess output is surfaced to the client via `engine_stderr` |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-35-09 | Tampering | tampered `job.Opts` row reaching ffmpeg | mitigate | `convert.AVOptsFromMap` strict re-parse before `MarkActive`; failure is terminal with `invalid_options`, mirroring the document/audio handlers' discipline |
| T-35-10 | Denial of Service | infinite retry of a hopeless job | mitigate | `isAVTerminal` returns true for every deterministic sentinel; terminal paths wrap `asynq.SkipRetry`; transcode-timeout retries are bounded by `AV_MAX_RETRY=2` |
| T-35-11 | Information Disclosure | `engine_stderr` leaking server paths | accept | Same exposure the four existing engine classes already accept; paths are server-generated temp dirs, and clients are internal services only |
| T-35-12 | Repudiation | double-counted job-outcome metrics on retry | mitigate | The transient path records no outcome and calls no `MarkFailed`, matching the existing handlers |
| T-35-SC | Tampering | npm/pip/cargo installs | mitigate | N/A — zero dependencies added |
</threat_model>
