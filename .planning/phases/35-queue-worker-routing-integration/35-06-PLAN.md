<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| untrusted client bytes → SniffVideo's EBML parser | A previously dead-code bounded parser goes live on the upload path for the first time (D-08) |
| untrusted client bytes → AVConverter subprocess path | Registration means client-controlled containers reach ffmpeg/ffprobe through a newly reachable entry point |
| client `opts` JSON → ffmpeg argv | AVOpts values influence encoder settings and seek timecodes |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-35-21 | Elevation of Privilege | `AVConverter.Convert` reachable from the network for the first time | mitigate | No new call site is created — `Convert` is the exact unit audited in 34-SECURITY.md (17/17 threats closed, ASVS 2), including `-protocol_whitelist file,crypto` + `-nostdin` + `file:`-prefixed paths on every ffmpeg/ffprobe invocation. Verify AVE-02 non-regression by asserting `grep -c protocol_whitelist internal/convert/av.go` is unchanged and stating the count in the SUMMARY |
| T-35-22 | Denial of Service | `SniffVideo`'s EBML parser on hostile input | mitigate | The parser is bounded-peek by construction (AVE-01, tested in Phase 34) and runs after the global 2 GiB `MaxBytesReader` bound; it is placed before `SniffAudio` so it does not add to the chain's worst-case buffering |
| T-35-23 | Tampering | client opts reaching ffmpeg argv | mitigate | `ParseAVOpts` closed-allowlist (`checkStrictObject`) + `ValidateAVApplicability` at create time; Phase 34's injection test proves client bytes never reach argv. No clamping/coercion added at this layer |
| T-35-24 | Tampering | registry pair collision silently misrouting jobs | mitigate | `TestAVAudioPairDisjointness` (plan 01) re-run after registration; disjoint target sets make collision impossible by construction today |
| T-35-25 | Denial of Service | generic-brand ISOBMFF audio-only file misrouted to the av engine (IN-01) | mitigate | `ErrAVNoVideoStream` (plan 01) makes it terminal with a `no_video_stream` error code (plan 03) instead of a confusing retry loop |
| T-35-SC | Tampering | npm/pip/cargo installs | mitigate | N/A — zero dependencies added |
