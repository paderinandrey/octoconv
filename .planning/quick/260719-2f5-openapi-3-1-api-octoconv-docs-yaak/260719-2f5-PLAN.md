ROUTES (internal/api/routes.go):
- GET  /healthz                    — PUBLIC, no auth, no /v1 prefix
- POST /v1/jobs                    — auth required (multipart create)
- GET  /v1/jobs/{id}               — auth required
- GET  /v1/formats                 — auth required
- POST   /v1/presets               — auth required
- GET    /v1/presets               — auth required (?all=true includes inactive)
- GET    /v1/presets/{name}        — auth required
- PUT    /v1/presets/{name}        — auth required
- DELETE /v1/presets/{name}        — auth required
- POST/GET/GET/PUT/DELETE /v1/system/presets[/{name}] — auth + OPERATOR ONLY
  (requireOperator; a non-operator caller gets the uniform 404, never 403)

Middleware on /v1: ratelimit.ByIP -> auth.Middleware -> ratelimit.PerClient.
Rate-limit rejection surfaces as HTTP 429 on every /v1 route.

AUTH (internal/auth): header "Authorization: ApiKey <key>". ParseAPIKey splits
on whitespace, requires exactly 2 fields, scheme token "ApiKey" (case-insensitive
via EqualFold). Missing/malformed -> 401 {"error":"missing or malformed
Authorization header"}. Unknown/revoked key -> 401 {"error":"invalid api key"}.

ERROR BODY SHAPE (writeError, everywhere): {"error": "<message>"} — a single
string field named "error". JSON responses are application/json.

POST /v1/jobs (handleCreateJob) — multipart/form-data fields:
- file         (required, binary) — the upload
- target       (string) — destination format, e.g. "json", "pdf", "webp"
- opts         (string) — a JSON STRING (engine-keyed; audio opts below)
- callback_url (optional string) — SSRF-validated webhook URL
- preset       (optional string) — named server preset; MUTUALLY EXCLUSIVE with
                 target/opts (sending both -> 422 "specify either preset or
                 target/opts, not both")
Success: 202 Accepted, body {"job_id": "<uuid>", "status": "queued"} and,
when opts were applied, an "opts" object echoing the normalized opts.
Errors (all body {"error":"..."}):
  400 — invalid multipart form / missing file / missing target format /
        cannot determine source format from filename / invalid callback_url /
        invalid preset name
  401 — auth (see above)
  413 — "file exceeds size limit" (MaxBytesReader)
  422 — unsupported conversion: <from> -> <to> / declared format X does not
        match detected content Y / unrecognized file content for <name> /
        invalid opts / opts not applicable to this conversion / unknown or
        inactive preset / macro-carrying documents are not accepted /
        password-protected Office file is not supported / legacy binary Office
        format ... / declared uncompressed size exceeds configured limit /
        declared image dimensions exceed configured limit
  429 — rate limited
  500 — internal (store/create/enqueue failures)

GET /v1/jobs/{id} (handleGetJob):
- 200 body: {"job_id","status", "opts"?, and by status:}
    status "done"   -> adds "download_url" (presigned S3 GET URL)
    status "failed" -> adds "error_code" and/or "error_message"
- statuses: queued | active | done | failed
- known error_code value: "duration_exceeded" (audio; error_message
  "declared audio duration exceeds the configured maximum")
- 400 "invalid job id"; 404 "job not found" (also returned for a job owned by a
  DIFFERENT client — cross-client access is indistinguishable from not-found)

GET /v1/formats (handleListFormats): 200 body
  {"engines": {"<class>": {"pairs": [["<from>","<to>"], ...]}, ...}}
  classes: image, document, html, audio.

SUPPORTED PAIRS (convert registry — for /v1/formats example + docs):
- image (png,jpg,webp,heic,tiff): every ordered pair where from!=to (20 pairs)
- document: docx,odt,xlsx,ods,pptx,odp -> pdf; plus docx<->odt, xlsx<->ods,
  pptx<->odp (6 to-pdf + 6 cross = 12 pairs)
- html: html -> pdf
- audio ({mp3,wav,m4a,ogg} x {txt,srt,vtt,json}): 16 pairs

AUDIO opts (convert.AudioOpts, JSON): {"language": "<v>", "translate": <bool>}
  language allowlist: auto,en,ru,es,fr,de (omit => whisper auto). target=json
  yields segment+word timestamps; txt/srt/vtt are plain/subtitle outputs.

PRESET WRITE DTO (presetRequest): {"name","target_format","options","description"}
PRESET READ DTO (presetResponse): {"name","version","scope","operation",
  "target_format","options","description","is_active","created_at","updated_at"}
  scope is "user" or "system" (system = read-only via this API).
  POST /v1/presets: 201 presetResponse; 409 "preset already exists";
  400 "invalid preset name"/"invalid JSON body"; 422 "invalid opts".
  GET /v1/presets: 200 array of presetResponse (?all=true includes inactive).
  GET/PUT/DELETE /v1/presets/{name}: 200; 404 "preset not found" (nonexistent,
  cross-client, and system-scope-write all collapse to this).
  DELETE returns 200 {"name":"<name>","is_active":false}.
  /v1/system/presets/* mirror these shapes but are operator-only.

healthz body: {"postgres":"ok","redis":"ok","s3":"ok","status":"ok"} on 200;
on 503 the per-dep value may be "unreachable" and "status":"degraded".

Structure:
- Header comment (YAML `#`) at the very top: "OctoConv API — OpenAPI 3.1. Import
  into Yaak: Import -> OpenAPI -> docs/openapi.yaml".
- `openapi: 3.1.0`; `info` with title "OctoConv API", a version string, and a

- `servers`: a single entry `url: "{base_url}"` with `variables.base_url.default:
  "http://localhost:8090"` and a description. This is the requested base_url
  server variable.
- `components.securitySchemes`: define `ApiKeyAuth` as
  `type: apiKey`, `in: header`, `name: Authorization`, with a `description` that
  explicitly states the value format is `ApiKey <key>` (the literal "ApiKey "
  prefix followed by the key). Model it as `type: apiKey` (NOT `type: http,
  scheme: apikey`, which does not validate and does not import cleanly). Document
  the prefix in the description since apiKey schemes cannot express it structurally.
- Top-level `security: [{ ApiKeyAuth: [] }]` so every operation defaults to
  requiring the key. On `GET /healthz` set `security: []` to make it public.
- `components.schemas`: define reusable schemas — `Error` ({error: string}),
  `JobCreated`, `JobStatus` (with status enum queued|active|done|failed and the
  optional download_url/error_code/error_message/opts fields), `AudioOpts`
  (language enum auto|en|ru|es|fr|de + translate boolean), `FormatsResponse`,
  `PresetRequest`, `PresetResponse`, `Health`. Reference them from operations
  via `$ref`.
- `paths` — one entry per route in `<facts>` ROUTES, with correct HTTP methods,
  path params ({id}, {name}), and per-operation `responses` using the real
  status codes and the `Error` schema for failures. Tag operations (Jobs,
  Formats, Presets, System, Health) for readable Yaak grouping.
  - `POST /v1/jobs`: `requestBody` `content: multipart/form-data` with a schema
    whose properties are file (type string, format binary), target, opts (string
    — a JSON string), callback_url, preset; mark file required. Add an
    `encoding` note if helpful. Include an AUDIO **example** on the requestBody

    second example showing target=json for word/segment timestamps. Document
    responses 202/400/401/413/422/429/500.
  - `GET /v1/jobs/{id}`: 200 JobStatus (show done-with-download_url and
    failed-with-error_code examples, including error_code "duration_exceeded"),
    400/401/404/429.
  - `GET /v1/formats`: 200 FormatsResponse with an example enumerating the
